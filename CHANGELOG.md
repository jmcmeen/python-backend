# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.0] - 2026-06-01

A "production-readiness" pass on the original blueprint. Toolchain modernized to `uv` + `pyproject.toml`; Docker + Postgres added; transactional boundary tightened; observability and CI gates introduced.

### Added

- `pyproject.toml` (PEP 621 metadata + PEP 735 `[dependency-groups]`) and `uv.lock` for reproducible installs.
- `.python-version` pinning Python 3.13.
- `ruff` configuration (lint + format, replaces black/isort/flake8) and `.pre-commit-config.yaml` with ruff, uv-lock, and hygiene hooks.
- `mypy` strict configuration scoped to `app/`.
- `pytest-cov` with an 80% coverage floor.
- `Makefile` with `help`, `install`, `dev-setup`, `dev`, `test`, `test-unit`, `test-integration`, `lint`, `fmt`, `typecheck`, `migrate`, `migration`, `docker-build`, `docker-up`, `docker-down`, `clean`.
- Multi-stage `Dockerfile` using the official `uv` image as a layer, with a BuildKit cache mount for the uv cache.
- `docker/entrypoint.sh` that retries `alembic upgrade head` before launching the app process.
- `docker-compose.yml` with Postgres 17 (healthchecked) + app service.
- `.dockerignore` and `.env.example`.
- `.github/workflows/ci.yml`: lint job + test matrix (`sqlite` and `postgres`) using `astral-sh/setup-uv@v4` with caching.
- `/health` (liveness) and `/ready` (DB `SELECT 1`) endpoints, mounted at the root.
- `app/core/logging.py`: `structlog` configuration (JSON for prod, console for dev).
- `app/core/middleware.py`: `RequestIdMiddleware` reads/generates `X-Request-ID`, binds it via `structlog.contextvars`, echoes it on the response, and emits a structured per-request access log with method, path, status, and duration.
- `Settings.cors_origins`, `log_level`, `log_format` fields.
- Field validator on `Settings.secret_key` that rejects placeholder values (`replace-with` / `change-me`).
- Initial Alembic migration (`migrations/versions/e0974a44791f_initial.py`) committed to the repo.
- `dispose_engine()` helper called from the lifespan shutdown hook.
- New tests: `tests/unit/test_config.py` (placeholder rejection + real secret), `tests/integration/test_health.py` (health/ready + request-ID echo/generation).

### Changed

- `app/core/database.py`: `get_db` now commits on success and rolls back on exception. The session is the transactional boundary, not the repository.
- `app/repositories/user_repo.py`: `create()` calls `flush()` only — no `commit()`, no `refresh()`.
- `app/core/security.py`: migrated from `python-jose` to `PyJWT`; same function signatures.
- `app/core/config.py`: `secret_key` is now `SecretStr` with no default; default `database_url` switched to `postgresql+asyncpg://...`; `extra="ignore"` set so the `.env` file can share `POSTGRES_*` variables with docker-compose without breaking the loader.
- `app/main.py`: dropped `Base.metadata.create_all` from the lifespan (Alembic owns the schema); added `RequestIdMiddleware`; CORS now reads from `settings.cors_origins`; `configure_logging()` is called at import time; `health.router` mounted at `/`.
- `app/api/v1/dependencies.py` and `app/api/v1/endpoints/user.py`: converted from `Depends(...)` default arguments to the modern `Annotated[Type, Depends(...)]` pattern; reusable type aliases (`DbSession`, `UserServiceDep`, etc.) live in `dependencies.py`.
- `migrations/env.py`: rewritten for fully async migrations using `async_engine_from_config`. Reads `settings.database_url` directly (no `+driver` suffix stripping).
- `migrations/script.py.mako`: updated to use modern `str | None` / `collections.abc.Sequence` so future autogenerated migrations are ruff-clean.
- `alembic.ini`: removed the static `sqlalchemy.url` line — `env.py` sets it from settings.
- `tests/conftest.py`: `TEST_DATABASE_URL` is read from `DATABASE_URL` env (so CI can run the same suite against Postgres); the `get_db` override is wrapped in an `autouse=True, scope="session"` fixture; `override_get_db` mirrors the production commit-on-success boundary; `SECRET_KEY` / `DATABASE_URL` / `LOG_LEVEL` are set at the top of the file before any app imports.
- `README.md`: rewritten around `uv` / Docker / Make; added the make-target table; noted the upcoming companion blog post.
- `CLAUDE.md`: extended with a "Modernized in v0.2" section documenting the new conventions (uv, ruff, the UoW boundary, async Alembic, structlog, the `Annotated` dependency pattern).
- `.gitignore`: added `*.db`, `*.db-journal`, and `.env.local`.

### Removed

- `requirements.txt` (replaced by `pyproject.toml` + `uv.lock`).
- `pytest.ini` (config now lives in `[tool.pytest.ini_options]`).
- `test.db` (was previously committed by accident; covered by the new `*.db` ignore).
- `python-jose` and `python-jose[cryptography]` (unmaintained).
- `Base.metadata.create_all` call from the application lifespan (Alembic is the schema source of truth).
- Hard-coded `secret_key` default (placeholder secrets are now refused at import).

### Security

- Removed `python-jose` — last release 2023-12, CVE-2024-33663 unresolved.
- Settings now refuse to load with a placeholder `SECRET_KEY` containing `replace-with` or `change-me`.
- CORS no longer combines `allow_origins=["*"]` with `allow_credentials=True` (browser-rejected combination).
- Container runs as a non-root `app` user.

## [0.1.0] - 2026-04-03

Initial release of the layered FastAPI backend template. Companion to [The Architecture Blueprint Every Python Backend Project Needs](https://medium.com/the-pythonworld/the-architecture-blueprint-every-python-backend-project-needs-207216931123).

### Added

- Layered project structure: `app/{api,core,models,schemas,services,repositories,utils}/`.
- FastAPI app with a single API version (`/api/v1/`).
- User domain end-to-end:
  - `models/user.py` (SQLAlchemy `Mapped[]` syntax),
  - `schemas/user.py` (`UserCreate`, `UserRead`),
  - `services/user_service.py` (framework-independent business logic),
  - `repositories/user_repo.py` (data-access wrapper),
  - `api/v1/endpoints/user.py` (CRUD endpoints).
- `app/core/config.py`: Pydantic `BaseSettings` with `.env` loading.
- `app/core/database.py`: async SQLAlchemy engine + session factory + `get_db` dependency.
- `app/core/security.py`: JWT create/decode helpers (HS256 via `python-jose`); bcrypt password hashing in `app/utils/hashing.py`.
- `app/api/v1/dependencies.py`: DI wiring for repository and service.
- Async lifespan that ran `Base.metadata.create_all` at startup.
- Alembic configuration (`alembic.ini`, `migrations/env.py`, `migrations/script.py.mako`) — synchronous, with `+aiosqlite` driver suffix stripped at runtime.
- Pytest test suite:
  - `tests/unit/test_user_service.py` (service-level tests against an in-memory SQLite),
  - `tests/integration/test_user_api.py` (httpx-driven API tests),
  - `tests/conftest.py` with `db` and `client` fixtures.
- `requirements.txt` with unpinned upper-bounded versions.
- `pytest.ini` with `asyncio_mode = auto`.
- `README.md` describing the layered architecture.
- `CLAUDE.md` for AI coding agents.
- CC0 1.0 `LICENSE`.
