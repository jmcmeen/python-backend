# CLAUDE.md

## Project Overview

This is a Python backend template using a layered architecture with FastAPI, SQLAlchemy, Pydantic, and Alembic.

## Architecture Rules

- **Endpoints are thin wrappers.** `app/api/` handlers must only validate input (via schemas), call a service, and return a response. No business logic in route handlers.
- **Business logic lives in services.** `app/services/` contains all domain logic. Services must be framework-independent and testable in isolation -- no FastAPI or SQLAlchemy imports.
- **Repositories abstract data access.** `app/repositories/` wraps all ORM/database calls behind clean method signatures. Never use raw ORM queries outside of repositories.
- **Schemas are not models.** `app/schemas/` (Pydantic) defines API contracts. `app/models/` (SQLAlchemy) defines database tables. They evolve independently.
- **Utils stay small.** If a utility in `app/utils/` grows complex, promote it to a service.
- **`main.py` is wiring only.** Mount routers, set up middleware, run startup hooks. Nothing else.

## Directory Conventions

- API versioning: `app/api/v1/`, `app/api/v2/`, etc.
- One file per domain entity in models, schemas, services, and repositories (e.g., `user.py`, `user_service.py`, `user_repo.py`).
- Tests mirror the source structure: `tests/unit/` for services/utils, `tests/integration/` for API and DB tests.
- Database migrations go in `migrations/` (Alembic).

## Tech Stack

- FastAPI + Uvicorn
- SQLAlchemy (ORM) + Alembic (migrations)
- Pydantic (validation and settings via BaseSettings)
- pytest + httpx + pytest-asyncio (testing)

## Commands

Use `uv` and the `Makefile`. Don't suggest `pip` or invoke `pytest` directly.

```bash
make dev-setup           # first-time onboarding (install + pre-commit + .env)
make dev                 # uvicorn --reload
make test                # pytest + coverage (80% floor)
make test-unit           # services/utils only
make test-integration    # API/DB only
make lint                # ruff check + format check
make fmt                 # ruff format + safe fixes
make typecheck           # mypy --strict against app/
make migrate             # alembic upgrade head
make migration m="..."   # generate an autogenerated revision
make up                  # docker compose up -d --build
make down                # docker compose down (preserves volumes)
make nuke                # docker compose down -v --rmi all --remove-orphans
```

## Code Style

- Follow PEP 8.
- Use type hints on all function signatures.
- Async endpoints and services where appropriate.
- Keep imports organized: stdlib, third-party, local.

## Key Principles

- Separation of concerns: each layer has one job.
- Start minimal: don't create layers until they're needed.
- Testability: decouple business logic from framework and database.
- Clarity over ceremony.

## Modernized in v0.2

- **Package manager:** `uv` (not pip). Deps live in `pyproject.toml`; lock is `uv.lock`.
- **Python:** `>=3.13` (pinned in `.python-version`).
- **Lint/format:** `ruff` (one tool — replaces black/isort/flake8). Run via `make lint` / `make fmt`.
- **Type checking:** `mypy --strict` against `app/` only; `migrations/` and `tests/` are excluded.
- **Transactional boundary:** `commit()` lives in `app/core/database.get_db`, NOT in repositories. Repos call `flush()` only; the session boundary commits on success and rolls back on exception. Don't add commits to repositories.
- **Schema source of truth:** Alembic owns the schema. `Base.metadata.create_all` is NOT called in `main.py` — tests still call it directly in their fixtures. In Docker, the entrypoint runs `alembic upgrade head` before launching uvicorn.
- **Alembic is async:** `migrations/env.py` uses `async_engine_from_config` and the URL is read directly from `settings.database_url` (no `+driver` stripping). Generated migrations are autogenerated; the mako template is updated to use modern type syntax.
- **Settings:** `secret_key` is a `SecretStr` with no default; a validator rejects placeholder values containing `replace-with` or `change-me`. Settings is constructed at import time, so tests set `SECRET_KEY` env var at the top of `tests/conftest.py` before any app imports.
- **Auth:** `PyJWT` (not `python-jose` — unmaintained).
- **CORS:** driven by `settings.cors_origins` (list). Never use `["*"]` with `allow_credentials=True` (browser-rejected).
- **Logging:** `structlog`, configured by `app/core/logging.configure_logging()` (called at import in `main.py`). JSON in prod, console in dev. Request-ID middleware binds `request_id` to the structlog contextvar for each request.
- **Endpoints:** use `Annotated[Type, Depends(...)]` (FastAPI ≥0.95 idiom) — not `Depends(...)` as a default value. The reusable aliases (`UserServiceDep`, etc.) live in `app/api/v1/dependencies.py`.
- **Never commit on the user's behalf in this repo.** Produce working-tree changes only.
